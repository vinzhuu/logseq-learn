- ## 什么是比价
	- 消费者, 先在拼多多主站（app/小程序/H5）访问商品详情页.
	- 然后, 在短时间内再通过访问 **多多客推广链**接 购买该商品。
- ## 比价订单佣金
	- 比价订单佣金有两种说法:

- ## 如何判断是否为比价订单
	- ### 下单前
		- 需要 用户 uid 先绑定备案 (参见: [[多多进宝 - 授权备案]] )
		  logseq.order-list-type:: number
		- 后续调用 API 查询商品时, 传入用户在 **我方系统的 id (uid)**  , 获取比价标识.
		  logseq.order-list-type:: number
			- 出参 `predict_promotion_rate` = 0 即为 比价 .
		- ==如果用户没有事先绑定备案:==
			- 我们调用 多多进宝 API 传入 uid , 查询商品详情时, 将不知道是否触发拼多多 **比价判定** .
			- 因为, 我们系统中的用户 uid 没有绑定拼多多账号, 调用 API 时, 拼多多根本不知道这个 用户 有没有在拼多多主站浏览过商品.
			- ==此时, 进入拼多多时, 会要求进行授权, 授权成功后调 API 会发现触发了 **比价** .==
	- ### 下单后
		- 参考: [多多客比价订单规则](https://jinbao.pinduoduo.com/qa-system?questionId=296)
		- 多多进宝后台查看.
		  logseq.order-list-type:: number
			- 如果 **比价订单** 的状态为 **审核成功** , 则会被标注为 “比价订单” .
		- API 接口查询.
		  logseq.order-list-type:: number
			- 通过如下 API 查询订单.
				- `price_compare_status` = 0  : 非比价.
				- `price_compare_status` = 1  : 比价.
			- ``` sh
			  pdd.ddk.order.detail.get
			  pdd.ddk.order.list.increment.get
			  pdd.ddk.order.list.range.get
			  pdd.ddk.all.order.list.increment.get
			  pdd.ddk.oauth.order.detail.get
			  ```
- ## 分享给好友购买
	- 如下场景：
		- 用户 1001 在拼多多账号 A 浏览了商品甲。
		  logseq.order-list-type:: number
		- 用户 1001 在拼多多账号 A 复制链接进入返利 APP 粘贴。
		  logseq.order-list-type:: number
		- 返利 APP 识别商品，然后传入用户 A 的 uid 1001 调用 `pdd.ddk.goods.promotion.url.generate` 接口，得到转链。
		  logseq.order-list-type:: number
			- 若用户 1001 通过转链跳转到拼多多账号 A 购买， 则用户 1001 将没有返利。
			  logseq.order-list-type:: number
			- 若用户 1001 通过转链跳转到拼多多账号 B 购买， 则用户 1001 将得到返利。
			  logseq.order-list-type:: number
				- 比如，跳转到自己的其他账号，或者将转链发送给他人购买。
